# Дизайн: опциональный выбор режима сессии

**Дата:** 2026-05-07  
**Статус:** Approved

## Проблема

Сейчас при открытии приложения пользователь обязан сначала выучить 10 новых слов, и только потом приступить к повторению. Нет возможности пропустить новые слова и сразу перейти к повторению.

## Решение

На HomeScreen добавляются две отдельные кнопки вместо одной. Пользователь сам выбирает режим. После завершения одного режима возвращается на Home и при желании запускает второй.

---

## Секция 1 — HomeScreen

### Кнопки

| Кнопка | Условие показа |
|---|---|
| `📖 Новые слова (N)` | `newCount > 0` |
| `🔁 Повторение (N)` | `dueCount > 0` |
| `✓ На сегодня готово` | `newCount === 0 && dueCount === 0` |

### Стрик

Логика стрика (`lastStudyDate`, `streak`) переносится в оба обработчика без изменений. Проверка `lastStudyDate === today` защищает от двойного счёта если пользователь нажмёт обе кнопки за день.

### Что не меняется

`getSessionCards()` вызывается так же. Данные `newWords`, `dueCards`, `reservePool` получаются одним вызовом, как сейчас.

---

## Секция 2 — App.tsx навигация

### Колбэки

`onStartSession` разбивается на два:

```ts
onStartNew(newWords: Word[], reservePool: Word[])  // → screen: 'learning'
onStartReview(dueCards: Array<{ word: Word; card: CardProgress }>)  // → screen: 'quiz'
```

### Таблица переходов

| Действие | Откуда | Куда |
|---|---|---|
| Нажать "Новые слова" | home | learning |
| Нажать "Повторение" | home | quiz |
| Завершить все новые слова | learning | **home** (не quiz) |
| Нажать "← Выйти" (прервать) | learning | **home** |
| Завершить quiz | quiz | results |
| Нажать "Домой" | results | home |

**Ключевое изменение:** после `LearningScreen.onComplete` больше не переходим автоматически на quiz. Пользователь сам решает, нажимать ли "🔁 Повторение" на Home.

---

## Секция 3 — LearningScreen

### Новый prop

```ts
onExit: () => void
```

### Кнопка выхода

Небольшая кнопка "← Выйти" в левом верхнем углу экрана. При нажатии вызывает `onExit()`.

### Поведение при выходе

- Никаких дополнительных сохранений в IndexedDB при выходе
- Слова, по которым уже нажали ✓ Запомнил, уже записаны — остаются
- Непройденные слова остаются со статусом `new` и появятся снова при следующем старте
