# Design: "Ещё раз" — перенос слова в конец очереди

**Дата:** 2026-04-21  
**Статус:** Approved

## Проблема

Кнопка "Ещё раз" в `LearningScreen` сломана: `onClick={() => setIndex(index)}` устанавливает тот же индекс, React не перерисовывает ничего. Слово не повторяется и не перемещается.

## Решение

Заменить no-op на `handleAgain`, которая переносит текущее слово в конец массива `localWords`. Индекс остаётся прежним — следующее слово "въезжает" на текущую позицию.

## Реализация

**Файл:** `src/screens/LearningScreen.tsx`

Заменить:
```tsx
onClick={() => setIndex(index)}
```

На вызов новой функции:
```ts
function handleAgain() {
  const updatedWords = [
    ...localWords.slice(0, index),
    ...localWords.slice(index + 1),
    word,
  ]
  setLocalWords(updatedWords)
}
```

Никаких изменений в `index`, `db`, `types`, `sm2` не требуется.

## Граничные случаи

| Ситуация | Поведение |
|----------|-----------|
| Одно слово в списке | `updatedWords` идентичен — слово показывается снова |
| Слово последнее в списке | То же — показывается снова |
| Слово в середине | Следующее слово появляется сразу, перенесённое идёт в конец |

## Счётчик

`(index + 1 / localWords.length)` корректен — длина массива не меняется.
